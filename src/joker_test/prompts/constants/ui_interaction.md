UI 元素交互指南：

**按钮/菜单项**
- 有文字的按钮直接用界面元素给的坐标作为 click 的 x/y
- 纯图标按钮（无文字）需要从截图中视觉定位图标位置作为 click 坐标

**滑块（Slider）/ 进度条（Progress Bar）**
- 滑块由两部分组成：轨道（固定线条）和手柄（可拖动的圆形/方形控件）
- OCR 可能只识别到轨道旁的静态文字标签，**文字值可能不会随操作实时更新**
- 不能依赖 OCR 的数值文字判断滑块当前位置
- 需要从截图中**视觉定位手柄的图形位置**：手柄在轨道上的相对位置 = 当前实际值
- 拖拽（swipe）时：x/y 必须是手柄中心的坐标，不能是文字标签的位置
- 从手柄当前位置向目标方向拖拽，不是从轨道端点开始

**旋钮（Knob）/ 滚轮**
- 视觉定位旋钮中心，沿圆弧方向拖拽
- OCR 文字值仅供参考，以截图中的图形位置为准

**文本输入框**
- 先 click 输入框获取焦点，再用 type_text 输入内容

**页面切换**
- 使用 swipe 或 scroll 进行翻页/滚动
- 横向标签栏通常在底部或顶部，纯图标标签需从截图判断
